# BUGS

## BUG-001 — POST /api/1/item возвращает ответ, не соответствующий контракту

**Краткое описание:**  
При создании объявления ручка `POST /api/1/item` возвращает только поле `status` со строкой вида `Сохранили объявление - <uuid>` вместо объекта созданного объявления.

**Шаги воспроизведения:**
1. Отправить `POST https://qa-internship.avito.com/api/1/item`
2. Передать валидное тело запроса:

```json
{
  "sellerID": 123456,
  "name": "bug-check",
  "price": 12345,
  "statistics": {
    "likes": 1,
    "viewCount": 10,
    "contacts": 2
  }
}
```

3. Посмотреть статус и тело ответа.

**Фактический результат:**  
HTTP status: `200 OK`

Тело ответа:

```json
{
  "status": "Сохранили объявление - 6621ba5d-ab5c-482f-bab6-8a76454c7b52"
}
```

**Ожидаемый результат:**  
Ручка должна возвращать объект созданного объявления с полями:
- `id`
- `sellerId`
- `name`
- `price`
- `statistics`
- `createdAt`

**Серьёзность:**  
Major

**Окружение:**  
- Host: `https://qa-internship.avito.com`
- OS: Windows
- Python: 3.13.5
- pytest: 8.4.1
- Postman

## BUG-002 — GET /api/2/statistic/{id} возвращает неконсистентный статус для невалидного id

**Краткое описание:**  
При запросе статистики по невалидному `id` ручка `GET /api/2/statistic/{id}` возвращает HTTP `404`, но в теле ответа указывает статус `"400"` и сообщение о некорректном идентификаторе.

**Шаги воспроизведения:**
1. Отправить `GET https://qa-internship.avito.com/api/2/statistic/bad-id`
2. Посмотреть HTTP status и тело ответа.

**Фактический результат:**  
HTTP status: `404 Not Found`

Тело ответа:

```json
{
  "result": {
    "message": "передан некорректный идентификатор объявления",
    "messages": null
  },
  "status": "400"
}
```

**Ожидаемый результат:**  
Если передан некорректный идентификатор объявления, ручка должна возвращать:
- HTTP status `400 Bad Request`
- согласованное тело ответа со статусом `400`

**Серьёзность:**  
Major

**Окружение:**  
- Host: `https://qa-internship.avito.com`
- OS: Windows
- Python: 3.13.5
- pytest: 8.4.1
- Postman
